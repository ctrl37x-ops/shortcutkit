# 디자인 시스템 가이드

이 프로젝트는 **듀얼 디자인 시스템**을 사용합니다.
메인 페이지(공개)와 관리자 페이지를 명확히 구분하여, 각 영역에 맞는 디자인을 적용합니다.

---

## 1. 메인 페이지 영역 (공개 페이지)

**경로**: `src/app/(main)/`
**성격**: 미니멀, 깔끔, 신뢰감

### 컬러
| 용도 | CSS 변수 | 값 |
|------|----------|-----|
| 배경 | `--main-bg` | `#FFFFFF` |
| 텍스트 | `--main-text` | `#1A1A1A` |
| 서브 텍스트 | `--main-text-sub` | `#6B7280` |
| 포인트 | `--main-primary` | `#2563EB` |
| 포인트 호버 | `--main-primary-hover` | `#1D4ED8` |
| 보더 | `--main-border` | `#E5E7EB` |
| 섹션 배경 | `--main-section-bg` | `#F9FAFB` |

### 타이포그래피
- **폰트**: Pretendard (한글), Inter (영문)
- **제목 (h1)**: 2.5rem / bold / 1.2 line-height
- **부제목 (h2)**: 1.75rem / semibold / 1.3 line-height
- **본문**: 1rem / regular / 1.6 line-height
- **캡션**: 0.875rem / regular / 1.5 line-height

### 레이아웃 원칙
- 최대 폭: `max-w-5xl` (1024px)
- 섹션 간 여백: `py-20` 이상
- 좌우 패딩: `px-6` (모바일), `px-8` (데스크탑)
- 카드: `rounded-xl`, 미세한 그림자 (`shadow-sm`), 넉넉한 패딩 (`p-8`)

### 컴포넌트 스타일
- **버튼**: 라운드 (`rounded-lg`), 충분한 패딩 (`px-6 py-3`), 깔끔한 호버 효과
- **입력 필드**: 심플한 보더, 포커스 시 포인트 컬러 아웃라인
- **네비게이션**: 심플, 로고 + 메뉴 항목 최소화
- **푸터**: 간결한 정보, 저작권 표시

---

## 2. 관리자 영역 (대시보드)

**경로**: `src/app/(admin)/`
**성격**: 실용적, 데이터 중심, 효율적

### 컬러
| 용도 | CSS 변수 | 값 |
|------|----------|-----|
| 사이드바 배경 | `--admin-sidebar-bg` | `#111827` |
| 사이드바 텍스트 | `--admin-sidebar-text` | `#D1D5DB` |
| 콘텐츠 배경 | `--admin-content-bg` | `#F3F4F6` |
| 카드 배경 | `--admin-card-bg` | `#FFFFFF` |
| 텍스트 | `--admin-text` | `#111827` |
| 포인트 | `--admin-primary` | `#3B82F6` |
| 성공 | `--admin-success` | `#10B981` |
| 경고 | `--admin-warning` | `#F59E0B` |
| 위험 | `--admin-danger` | `#EF4444` |

### 레이아웃
```
┌──────────────────────────────────────────┐
│  헤더 (검색, 알림, 프로필)                  │
├────────┬─────────────────────────────────┤
│        │                                 │
│ 사이드  │       콘텐츠 영역                 │
│  바    │                                 │
│ (240px)│   ┌─────────┐ ┌─────────┐      │
│        │   │  카드 1   │ │  카드 2   │      │
│  메뉴   │   └─────────┘ └─────────┘      │
│  항목   │                                 │
│        │   ┌─────────────────────┐      │
│        │   │    데이터 테이블       │      │
│        │   └─────────────────────┘      │
└────────┴─────────────────────────────────┘
```

- **사이드바**: 고정 240px, 다크 배경, 아이콘 + 텍스트 메뉴
- **헤더**: 높이 64px, 검색바 + 알림 + 사용자 프로필
- **콘텐츠**: 좌측 사이드바 제외한 나머지 영역, `p-6` 패딩
- **카드**: `rounded-lg`, `shadow-sm`, `p-6`

### 컴포넌트 스타일
- **데이터 테이블**: 스트라이프 행, 호버 효과, 페이지네이션
- **통계 카드**: 아이콘 + 수치 + 변화량 표시
- **폼**: 라벨 위 배치, 검증 메시지 인라인
- **모달**: shadcn/ui Dialog 컴포넌트 활용
- **토스트**: 우측 상단, 자동 닫힘

## 3. 공통 규칙

### 반응형 브레이크포인트
- **모바일**: < 768px
- **태블릿**: 768px ~ 1024px
- **데스크탑**: > 1024px
- 관리자 영역은 태블릿 이상에서만 사이드바 표시, 모바일에서는 햄버거 메뉴

### 접근성
- 포커스 링 필수 (`focus-visible:ring-2`)
- 최소 터치 타겟 44x44px
- 색상 대비 WCAG AA 기준 충족
- 시맨틱 HTML 태그 사용

### 애니메이션
- 기본 트랜지션: `transition-all duration-200 ease-in-out`
- 호버/포커스에만 적용, 불필요한 애니메이션 지양
- `prefers-reduced-motion` 미디어 쿼리 존중

### 다크모드
- 현재 버전에서는 지원하지 않음
- 추후 확장 가능하도록 CSS 변수 기반으로 설계
